骑士的移动（Knight Moves, UVa 439）

输入标准 8*8 国际象棋棋盘上的两个格子（列用 a~h 表示，行用 1~8 表示），求马最少需要多少步从起点跳到终点。例如从 a1 到 b2 需要四步。马的移动方式如下图所示。

样例输入：

e2 e4
a1 b2
b2 c3
a1 h8
a1 h7
h8 a1
b1 c3
f6 f6

样例输出：

To get from e2 to e4 takes 2 knight moves.
To get from a1 to b2 takes 4 knight moves.
To get from b2 to c3 takes 2 knight moves.
To get from a1 to h8 takes 6 knight moves.
To get from a1 to h7 takes 5 knight moves.
To get from h8 to a1 takes 6 knight moves.
To get from b1 to c3 takes 1 knight moves.
To get from f6 to f6 takes 0 knight moves.

题外话：还记得小时候老爹教我国际象棋的时候告诉我 —— 马是八面威风。

这道题目说起来也是有意思。作为算法初心者，我拿到这道题目的时候第一感觉是：这道题目要用 dfs 解决。如果你知道答案，先别笑，听我慢慢分析。

首先设定棋盘大小：

const int maxn = 8;

接着编写便于计算的结构体 Node，以及将输入的标识符转化为 Node 的方法 id_to_node：

struct Node {
  int r, c;
  Node(int r = 0, int c = 0): r(r), c(c) {}
};

Node id_to_node(const char* s) {
  return Node(s[0] - 'a', s[1] - '1');
}

添加判断边界条件的方法：

bool inside(Node u) {
  return u.r >= 0 && u.r < maxn && u.c >= 0 && u.c < maxn;
}

仔细观察上面图中马的移动方式，总结规律：

如果马的初始坐标为 (r, c)，则马可以移动到的位置（不考虑边界条件）有：(r+2, c-1), (r+2, c+1), (r+1, c+2), (r-1, c+2), (r-2, c+1), (r-2, c-1), (r-1, c-2), (r+1, c-2)

由此设定马下一步能移动的常量：

const int moves[8][2] = {{2, -1}, {2, 1}, {1, 2}, {-1, 2}, {-2, 1}, {-2, -1}, {-1, -2}, {1, -2}};

添加全局 Node 类型的变量 u0 和 u1，用于存储变换过后的初始状态和最终状态；以及全局二维整形数组变量 m，用于存储从初始状态至 (r, c) 的最小步数（m 中每个元素的初始值设为 -1，表示从初始状态起，还没有经过 (r, c) 这个点）：

Node u0, u1;
int m[maxn][maxn];

接着就要进入搜索的正题了，按照先入为主的第一感觉我采用了深度优先搜索：

void dfs(Node u, int last_move) {
  if (!inside(u)) {
    return; // out of bound
  }

  if (m[u.r][u.c] > -1) {
    return; // already visited, no need to continue
  } else {
    m[u.r][u.c] = last_move;
  }

  if (u.r == u1.r && u.c == u1.c) {
    return; // finished status
  }

  last_move += 1;

  for (int i = 0; i < 8; ++i) {
    dfs(Node(u.r + moves[i][0], u.c + moves[i][1]), last_move);
  }
}

其中参数 u 表示正在遍历的节点，last_move 表示遍历到当前节点已经花费的步数。

对 dfs 中三个不同的返回情况做下解释：第一种越界的情况是非法的，肯定不能继续运算了；最后一种找到最终结果之后，也不用继续计算了；第二种我当时的想法是，如果一个点已经被访问过了，那表示有其他搜索已经遍历过接下来的所有情况了，也就没有必要继续遍历了。

补全 main 函数：

int main(void) {
  char s0[3], s1[3];
  while (scanf("%s%s", s0, s1) == 2) {
    u0 = id_to_node(s0);
    u1 = id_to_node(s1);
    memset(m, -1, sizeof(m));

    dfs(u0, 0);
    printf("To get from %s to %s takes %d knight moves.\n", s0, s1, m[u1.r][u1.c]);
  }

  return 0;
}

编译并输入测试数据运行：

尼玛，哪里出问题了？！

检查并思考之后发现，发现上面的 dfs 中的第二种返回情况的依据：“如果一个点已经被访问过了，那表示有其他搜索已经遍历过接下来的所有情况了，也就没有必要继续遍历了”，是错误的。

原因是，当一个点被访问过的时候，已经访问的遍历方式并不一定是最优的解。

也就是说，在深度优先搜索的前提条件下，即使一个点已经被访问过了，那接下来还是要继续访问。

如果只是去掉第二个返回条件，再次编译运行后会出现 segmentation fault，没有终止条件是不行的。

在代码中限定最大搜索次数：

const int max_try = maxn;

并修改第二个返回条件为：

if (last_move > max_try) {
  return; // exceed max try
}

if (m[u.r][u.c] > -1) {
  m[u.r][u.c] = std::min(m[u.r][u.c], last_move);
} else {
  m[u.r][u.c] = last_move;
}

编译运行测试数据，结果正确。

这就完了？并不是，尝试提交答案，结果打脸了：

超时！

我横竖不甘心，上网尝试搜索答案，都不用仔细看，所有的解法都写着 bfs。看来先入为主的第一感觉不一定是正确的。QAQ

修改 dfs 为 bfs 结果如下：

void bfs(void) {
  std::queue<Node> q;

  q.push(u0);
  m[u0.r][u0.c] = 0;

  while (!q.empty() && m[u1.r][u1.c] == -1) {
    Node u = q.front();
    q.pop();

    for (int i = 0; i < 8; ++i) {
      Node next = Node(u.r + moves[i][0], u.c + moves[i][1]);
      if (inside(next) && m[next.r][next.c] == -1) {
        q.push(next);
        m[next.r][next.c] = m[u.r][u.c] + 1;
      }
    }
  }
}

注意这里的往搜索队列 q 中 push 节点的判断条件：inside(next) && m[next.r][next.c] == -1 —— 没有越界并且没有被访问过。

为什么 dfs 中节点没有被访问过不能被当做边界条件，但是 bfs 可以呢？

因为广度优先搜索的顺序是按照距离根节点的顺序进行的，也就是说：在广度优先搜索的前提下，如果一个节点已经被访问过了，那么之前访问这个节点的遍历方式一定优于这次访问这个节点的遍历方式。

再次尝试提交代码：

最终完整代码：

#include<cstdio>
#include<cstring>
#include<queue>

const int maxn = 8;

struct Node {
  int r, c;
  Node(int r = 0, int c = 0): r(r), c(c) {}
};

Node id_to_node(const char* s) {
  return Node(s[0] - 'a', s[1] - '1');
}

bool inside(Node u) {
  return u.r >= 0 && u.r < maxn && u.c >= 0 && u.c < maxn;
}

const int moves[8][2] = {{2, -1}, {2, 1}, {1, 2}, {-1, 2}, {-2, 1}, {-2, -1}, {-1, -2}, {1, -2}};

Node u0, u1;
int m[maxn][maxn];

void bfs(void) {
  std::queue<Node> q;

  q.push(u0);
  m[u0.r][u0.c] = 0;

  while (!q.empty() && m[u1.r][u1.c] == -1) {
    Node u = q.front();
    q.pop();

    for (int i = 0; i < 8; ++i) {
      Node next = Node(u.r + moves[i][0], u.c + moves[i][1]);
      if (inside(next) && m[next.r][next.c] == -1) {
        q.push(next);
        m[next.r][next.c] = m[u.r][u.c] + 1;
      }
    }
  }
}

int main(void) {
  char s0[3], s1[3];
  while (scanf("%s%s", s0, s1) == 2) {
    u0 = id_to_node(s0);
    u1 = id_to_node(s1);
    memset(m, -1, sizeof(m));

    bfs();
    printf("To get from %s to %s takes %d knight moves.\n", s0, s1, m[u1.r][u1.c]);
  }

  return 0;
}
